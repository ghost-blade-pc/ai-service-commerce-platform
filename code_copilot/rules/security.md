# 安全与配置规则

## 敏感信息

- 不写入真实密码、token、私钥、支付密钥、签名密钥、云服务凭证。
- 不在日志中输出完整手机号、身份证、支付渠道原始回调、access token、JWT、Cookie、签名串。
- 示例配置使用 `${ENV_NAME}` 或明显的占位值。

## 接口与鉴权

- 新增或修改公开 API 时，Spec 必须说明鉴权、签名、权限、参数校验、错误响应和兼容性。
- 外部接口调用必须说明超时、重试、失败返回、空响应、非成功 code 的处理。
- 支付、退款、登录、回调接口必须优先检查验签、幂等和重放风险。

## 数据与消息

- 数据库结构、Mapper XML、DAO、PO、Repository 必须保持一致。
- 涉及数据修复、状态补偿、批量更新时，必须写明回滚或补偿方案。
- MQ 消费者必须考虑重复消费、乱序、重试、死信和幂等。
- 不在消息体中新增敏感字段，除非 Spec 说明必要性、脱敏方式和消费者影响。

## 本地环境

- Docker Compose、`application-*.yml`、构建脚本中的端口和账号只作为本地/测试线索，不视为生产事实。
- 修改 `docs/dev-ops/` 或 `application-*.yml` 前，先确认影响哪个子项目和哪个 profile。
